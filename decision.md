ActivityPub仮想リレープロキシ 設計書
1. プロジェクトの目的と基本方針

    目的: 少人数Mastodonサーバーのストレージ容量を節約しつつ、自身の関心に合致したにぎやかし（投稿）を外部から取り込む。

    基本方針:

        中身（JSON）の厳密なパースは避け、可能な限り素通しする（土管化）。

        アプリケーション自体は完全にステートレス（状態を持たない）とし、マイクロサービス的に振る舞う。

        状態管理、キュー、キャッシュはすべて外部ミドルウェア（Redis）に一任し、Goアプリ側の実装を極限までシンプルにする。

2. システムアーキテクチャ

Docker Composeを用いた2コンテナ構成を採用し、堅牢性と運用保守の容易さを両立する。

    Proxy App コンテナ (Go):

        HTTPリクエストの受付（受付層）と、バックグラウンドでの非同期処理（ワーカー層）を1つのバイナリ・1つのコンテナ内でGoroutineを用いて並行稼働させる。

    Redis コンテナ:

        受信キュー、送信キュー、重複排除キャッシュのすべての永続化と状態管理を担う。

3. データ処理とルーティング要件

ActivityPubの複雑な仕様に深入りせず、リソースと負荷を最小化するフィルタリングを行う。

    重複排除 (Deduplication):

        到達したActivityの id をRedisにTTL付き（例: 数十分〜数時間）で保存。

        既にキャッシュに存在する id が届いた場合は即座にDrop（破棄）し、Mastodon本体への負荷を防ぐ。

    Activity種別によるルーティング:

        Create (新規投稿): 設定したルール（キーワード、ハッシュタグ等）で評価し、条件を満たしたものだけを通過させる。

        Delete, Update, Undo など: 状態変化を伝えるActivityは、内容に関わらず無条件で通過させる。これによりMastodon本体にゴミが残るのを防ぐ（対象が存在しない場合はMastodon側が安全に無視するため問題ない）。

    署名の再付与 (カプセル化):

        通過したデータを Announce（ブースト）でラップし、プロキシ自身（仮想アクター）の秘密鍵を用いてHTTP Signatureを生成し、Mastodon本体へ転送する。

4. 技術スタックとライブラリ選定 (Go)

「ActivityPubの重厚なライブラリを使わず、いかに手抜きをして堅牢に作るか」を基準に選定。

    HTTPルーター: github.com/go-chi/chi/v5

        軽量かつミドルウェア（一括署名検証など）の構築が容易。

    ActivityPub署名 (HTTP Signatures): github.com/go-fed/httpsig

        暗号化署名の生成・検証用。Mastodon互換性が高い。

    JSONハンドリング: 標準ライブラリ (json.RawMessage / map[string]any)

        サードパーティのActivityPub構造体マッピングは使用しない。方言によるパースエラーを回避するため、必要なキーだけを抽出し残りはバイト列のまま扱う。

    Redisクライアント: github.com/redis/go-redis/v9

        重複排除のキャッシュ管理用（SET EX NX等を利用）。

    ジョブキュー (非同期ワーカー): github.com/hibiken/asynq

        Redisバックエンドのキュー管理。Mastodonへの確実な送信、リトライ、指数的バックオフ、Dead Letter Queue処理を担う。

    環境変数・設定管理: github.com/caarlos0/env/v10 / github.com/joho/godotenv

        設定ミスを防ぐ型安全なENV読み込み。

5. インフラ・セキュリティ管理 (The Twelve-Factor App)

    設定の外部化:

        サーバーのドメイン名、接続先URL、Mastodonのトークンなどはすべて環境変数（.env）からコンテナに注入する。コード内には一切ハードコードしない。

    秘密鍵の扱い:

        仮想アクター用の秘密鍵（PEM形式）は、改行コードによるパースエラーを防ぐため Base64 でエンコードして1行の文字列にした上で環境変数にセットし、Goアプリ起動時にデコードしてメモリに展開する。
